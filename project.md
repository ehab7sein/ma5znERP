# تعليمات المشروع

أنت مهندس برمجيات Senior Full Stack وخبير في بناء أنظمة ERP وإدارة المخازن.

مهمتك هي إنشاء Web Application احترافي لإدارة مخزن أحذية باللغة العربية.

لا تستخدم أي Framework للواجهة مثل React أو Vue.

التقنيات المطلوبة:

* HTML5
* Tailwind CSS
* JavaScript ES6
* Node.js
* Express.js
* Supabase Database (لتخزين البيانات فقط)
* Local Authentication (بدون Supabase Auth)

---

# الهدف

إنشاء نظام بسيط وسريع لإدارة مخزن أحذية يعمل بالكامل باللغة العربية مع تصميم حديث Minimal وقابل للتوسع مستقبلاً.

---

# اللغة

* جميع النصوص باللغة العربية.

* جميع الرسائل والأزرار بالعربية.

# فكرة النظام

هذا النظام خاص بمخزن أحذية.

لا يوجد Barcode.

لا يوجد QR Code.

ولا يتم الاعتماد على Scanner.

يتم الاعتماد فقط على:

* اسم الموديل
* اللون
* المقاس

كل موديل يحتوي على عدة مقاسات.

كل مقاس له كمية مستقلة.

مثال

موديل

Nike Air

ألوان

أسود

المقاسات

39

40

41

42

43

44

لكل مقاس كمية مختلفة.

لا تقم بتخزين المقاسات داخل نفس جدول المنتجات.

أنشئ جدول مستقل للمقاسات.

---

# إدارة موديلات الأحذية

يمكن إضافة موديل جديد.

البيانات المطلوبة:

* اسم الموديل
* الفئة (رجالي - حريمي - أطفال - رياضي - كاجوال...)
* اللون
* الخامة
* الشركة أو العلامة التجارية
* ملاحظات

بعد إنشاء الموديل يمكن إضافة المقاسات الخاصة به.

كل مقاس يحتوي على:

* المقاس
* الكمية الحالية
* الحد الأدنى للتنبيه

# مواد التعبئة

أنشئ قسم مستقل باسم:

مواد التعبئة

ويحتوي على:

* علبة رجالي
* علبة حريمي
* علبة أطفال
* كيس
* استيكر
* ورق تغليف

كل عنصر يحتوي على:

* الاسم
* الكمية
* الحد الأدنى

يمكن:

* إضافة كمية
* صرف كمية
* تعديل الكمية
* حذف العنصر

وجميع العمليات تُسجل في سجل الحركات.

# سجل الحركات

يعرض الجدول:

* التاريخ
* الوقت

* نوع العنصر (حذاء / مواد تعبئة)
* اسم العنصر
* المقاس (إذا كان حذاء)
* الكمية
* الرصيد قبل العملية
* الرصيد بعد العملية
* المنفذ
* ملاحظات

يجب وجود:

* بحث
* فلترة بالتاريخ
* فلترة بنوع العملية
* ترتيب
